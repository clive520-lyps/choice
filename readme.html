系統部署與資料庫轉移指南

Firebase 的資料庫架構 Firebase 的資料庫架構

如何將「智慧測驗系統」綁定到您專屬的 Firebase 資料庫

1

建立 Firebase 專案

取得您專屬的雲端伺服器空間。

1. 前往 Firebase Console (控制台) 並登入您的 Google 帳號。

2. 點擊 「新增專案 (Add project)」。

3. 輸入專案名稱（例如：MySmartQuiz），點擊繼續。

4. 建議可關閉 Google Analytics (不強迫)，然後點擊 「建立專案 (Create project)」。

2

啟用驗證與資料庫服務

開啟老師登入權限與雲端資料儲存功能。

啟用 Authentication (登入驗證)

1. 在左側選單點擊 「建構 (Build)」 > 「Authentication」，然後點擊「開始使用」。

2. 選擇 「電子郵件/密碼 (Email/Password)」 提供者。

3. 將第一個選項（電子郵件/密碼）切換為 「啟用」，點擊儲存。

啟用 Firestore Database (資料庫)

1. 在左側選單點擊 「建構 (Build)」 > 「Firestore Database」，然後點擊「建立資料庫」。

2. 位置選擇 asia-east1 (台灣) 或任何您喜歡的伺服器位置，點擊下一步。

3. 安全性規則先選擇 「以測試模式啟動 (Start in test mode)」（注意：這允許所有人讀寫，正式上線後建議修改安全性規則），點擊建立。

3

取得專案金鑰 (firebaseConfig)

這是系統與資料庫連線的靈魂鑰匙。

1. 回到 Firebase 控制台首頁 (點擊左上角專案總覽)。

2. 點擊畫面中央的 「</>」 (網頁) 圖示 來新增一個網頁應用程式。

3. 隨便輸入一個應用程式暱稱 (如：QuizApp)，點擊「註冊應用程式」。

4. 註冊後，畫面會顯示一段程式碼，請找到裡面名叫 const firebaseConfig = { ... } 的區塊（如下圖範例所示）。請將這整個大括號的內容複製下來。

const firebaseConfig = {
  apiKey: "AIzaSyB9NJXQwdKweStTb...",
  authDomain: "your-project-id.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project-id.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcdefg"
};
4

修改系統原始碼 (替換金鑰)

最後一步，告訴您的網頁該去哪裡存取資料。

您需要修改兩個檔案：teacher.html 與 student.html。

請用任何文字編輯器 (如 VS Code、Sublime Text，甚至記事本) 打開這兩個檔案。

利用搜尋功能 (Ctrl+F) 尋找 const firebaseConfig，您會在大約開頭的 <script type="text/babel"> 區塊內找到它。

👉 將原本的 firebaseConfig 區塊，整段替換成您在【步驟三】複製的您專屬的設定碼。

修改 teacher.html

替換後，存檔。重新開啟網頁，您就能註冊成為您專屬資料庫的最高權限管理員（指揮官）。

修改 student.html

替換後，存檔。學生端就能正確讀取到您在老師端發布的班級名單與考卷任務了。

恭喜！系統已成功轉移完畢！

現在，您可以將這兩個 HTML 檔案放上任何支援靜態網頁的空間 (例如 GitHub Pages)，開始您的教學作戰了！
